Challenges and Lessons Learned with Jupyter Notebooks and GitHub Pages

Reflecting on the hurdles faced while building and sharing projects using Jupyter Notebooks and GitHub Pages.

1. Jupyter Notebook File Size Limitations

What I learned:

  • Managing large notebooks required extensive commit history as I made incremental changes to reduce file size and optimize performance.
  • Breaking notebooks into smaller chunks and generating outputs selectively helped keep repository sizes manageable.

What I would do differently:

  • I would automate file size reduction using nbconvert to generate lightweight HTML files before committing.
  • For large datasets, I’d store them externally and reference them instead of including them in the repository.

2. Rendering Jupyter Notebooks on GitHub Pages

What I learned:

  • Converting notebooks to HTML with nbconvert required template customization to maintain consistent styling across projects.
  • Keeping styling and interactivity uniform across multiple notebooks was challenging without a standardized workflow.

What I would do differently:

  • I’d develop a more streamlined template system to maintain consistency without excessive manual tweaks.
  • Exploring interactive alternatives like Binder or Voila could offer a more dynamic presentation of notebooks.

3. Integration with GitHub Pages and Jupyter Notebooks

What I learned:

  • Structuring the repository properly ensured that GitHub Pages could correctly serve notebooks and associated assets.
  • Frequent commits were necessary to track changes and manually update HTML versions of the notebooks.

What I would do differently:

  • Implement GitHub Actions to automate notebook-to-HTML conversion and deployment, reducing manual steps.
  • Leverage version control more effectively to ensure the live GitHub Pages site always reflects the latest changes.

4. Collaboration and Version Control

What I learned:

  • Maintaining clear, structured commit messages was crucial given the large number of incremental changes required.
  • Ensuring computational parts of notebooks were clean while using Markdown for documentation made collaboration easier.

What I would do differently:

  • Encourage more frequent commits to reduce the risk of merge conflicts when working with Jupyter Notebooks.
  • Explore using Git tools like nbdime to better visualize and manage notebook diffs.
